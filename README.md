# スロット設定判別ツール

パチスロの設定(設定1〜6)を、観測したボーナスからベイズ推定で判別するWebアプリです。
現在は「うみねこのなく頃に2」に対応しています。

## 使い方

1. 実機を打ちながら、ボーナスを引いたら**総ゲーム数(そのボーナスが成立した時点のもの)**・**契機**(何の役で成立したか)・**種別**(ボーナスの色/タイプ)を入力して「＋ ボーナスを追加」を押す
2. ボーナスを引くたびに繰り返す
3. 「設定判別」欄に各設定(1〜6)の確率がリアルタイムで表示される。観測数が増えるほど精度が上がる
4. 入力を間違えた場合は「入力履歴」から個別に削除、やり直したい場合は「全消去」

入力は上記の1本のフローだけでOKです。裏側では2種類の判別根拠を同時に計算しています。

- **契機×種別の内訳**による判別(例: 「確定役A」でWITCHボーナスが成立、など判別力の高い組み合わせ)
- **総ゲーム数あたりのBIG/REG当選回数**による判別(入力した総ゲーム数と契機×種別から自動集計)

総回転数や小役カウントを別途まとめて入力する必要はなく、ボーナスを引くたびに総ゲーム数だけ添えて記録すれば両方の判別に使われます。

### おすすめの使い方(うみねこ2は完走型ART)

うみねこ2は完走型ART(ARTに入ったら区切りが来るまで続く方式)のため、ART中に成立したボーナスは
「今何ゲーム目か」が分かりにくいことがあります。総ゲーム数の記録精度が判別精度に直結するので、
**本体オプションでWITCHランプを「一発告知」に設定し、当選した瞬間に総ゲーム数を確認して入力する**
使い方がおすすめです。これにより、ART中のボーナスも取りこぼさず正確に記録でき、判別の精度が上がります。

## セットアップ(開発者向け)

Node.js が必要です(このリポジトリは nvm での導入を想定しています)。

```bash
# 初回のみ: 依存パッケージのインストール
npm install

# 開発サーバの起動(ブラウザで http://localhost:5173 を開く)
npm run dev

# 本番ビルド(型チェック込み)
npm run build

# lint
npm run lint
```

nvm を使っている場合、シェルを開くたびに以下でNode/npmを読み込む必要があります。

```bash
export NVM_DIR="$HOME/.nvm"; \. "$NVM_DIR/nvm.sh"
```
