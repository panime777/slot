# スロット設定判別ツール

パチスロの設定(設定1〜6)を、観測したボーナスからベイズ推定で判別するWebアプリです。
現在は「うみねこのなく頃に2」に対応しています。

**公開URL**: https://slot-lyart.vercel.app

## 使い方

1. 実機を打ちながら、ボーナスを引いたら**当選ゲーム数**(そのボーナスが成立した時点の総ゲーム数)・**契機**(何の役で成立したか)・**種別**(ボーナスの色/タイプ)を入力して「＋ ボーナスを追加」を押す
2. ボーナスを引くたびに繰り返す
3. 「設定判別」欄に各設定(1〜6)の確率がリアルタイムで表示される。観測数が増えるほど精度が上がる
4. 入力を間違えた場合は「入力履歴」から個別に削除、やり直したい場合は「全消去」

入力は上記の1本のフローだけでOKです。裏側では2種類の判別根拠を同時に計算しています。

- **契機×種別の内訳**による判別(例: 「確定役A」でWITCHボーナスが成立、など判別力の高い組み合わせ)
- **総ゲーム数あたりのBIG/REG当選回数**による判別(入力した当選ゲーム数と契機×種別から自動集計)

総回転数や小役カウントを別途まとめて入力する必要はなく、ボーナスを引くたびに当選ゲーム数だけ添えて記録すれば両方の判別に使われます。

### 簡易設定推測(総ゲーム数のみ)

上記の詳細な記録とは別に、ページ下部の「簡易設定推測」では**総ゲーム数**・**BIG回数**・**REG回数**だけを
入力して大まかに判定できます。1件ずつボーナスを記録する時間がないときや、途中経過をざっくり知りたいときに使えます。

### おすすめの使い方

うみねこ2は完走型ART(ARTに入ったら区切りが来るまで続く方式)のため、ART中に成立したボーナスは何ゲーム数が分かりにくいことがあります。総ゲーム数の記録精度が判別精度に直結するので、
**WITCHランプによる告知を使用することで、当選ゲーム数を正確にすることができます（すり抜けもありますが...）**
これにより、ART中のボーナスも取りこぼさず正確に記録でき、判別の精度が上がります。

## セットアップ(開発者向け)

Node.js が必要です(このリポジトリは nvm での導入を想定しています)。

```bash
# 初回のみ: 依存パッケージのインストール
npm install

# 開発サーバの起動(ブラウザで http://localhost:5173 を開く)
npm run dev

# 本番ビルド(型チェック込み)
npm run build

# lint
npm run lint
```

nvm を使っている場合、シェルを開くたびに以下でNode/npmを読み込む必要があります。

```bash
export NVM_DIR="$HOME/.nvm"; \. "$NVM_DIR/nvm.sh"
```

## デプロイ

Vercelにデプロイ済みです(プロジェクト: `panime/slot`)。GitHubリポジトリと連携済みのため、
`master` に push すると自動的にビルド・デプロイされ、公開URLに反映されます。手動デプロイは不要です。

## 仕組み・データについて

設計や判別モデルの詳細は [CLAUDE.md](CLAUDE.md) にまとめています。
機種を追加したい場合は `src/machines/` にデータファイルを1つ足すだけで対応できます。

### データの出典

「うみねこのなく頃に2」の設定判別データ(契機×種別の複合確率、BIG/REG合算確率など)は、
以下の公開スペック情報を基にしています。詳細は [src/machines/umineko2.ts](src/machines/umineko2.ts) のコメントを参照してください。

- なな徹: https://nana-press.com/kaiseki/machine/1089/34695/
- DMMぱちタウン: https://p-town.dmm.com/machines/4925

なお、出典に具体値の記載がなく仮の値を当てている項目(REGの一部の列など)がある点も
`umineko2.ts` にコメントで明記しています。
